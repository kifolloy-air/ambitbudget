# Google Play — Package Identity

Sibling to `STORE_IDENTITY.md` (Microsoft Store). These values pin the app's
identity on Google Play so a future native-Android build (Tauri, Capacitor,
Bubblewrap, or anything else) can update the existing listing instead of
publishing a parallel one.

**A mismatched applicationId or signing key fingerprint means Play treats the
upload as a different app**, which would break updates for every existing
install and likely fail review. If you only remember one thing from this
file: don't change the applicationId, and don't lose the keystore.

### How to read the fingerprints later

```powershell
# Upload key (from the .keystore file PWABuilder emails you)
$keytool = "$env:JAVA_HOME\bin\keytool.exe"   # or any installed JDK keytool
& $keytool -list -v -keystore "<your>.keystore" -alias <alias> | Select-String "SHA256:"
```

```text
# App signing key (from Play Console)
Play Console → your app → Setup → App integrity → App signing
  → "App signing key certificate" → SHA-256 certificate fingerprint
```

Update this file with both fingerprints once Play has issued the App Signing
key (that happens on first upload, before production release).

## assetlinks.json — REQUIRED for TWA

The TWA only opens fullscreen (no Chrome URL bar) if a Digital Asset Links
file at `https://ambitbudget.com/.well-known/assetlinks.json` lists both
SHA-256 fingerprints above. PWABuilder generates this file for you.

```json
[{
  "relation": ["delegate_permission/common.handle_all_urls"],
  "target": {
    "namespace": "android_app",
    "package_name": "com.kifolloyair.ambitbudget",
    "sha256_cert_fingerprints": [
      "<upload key SHA-256>",
      "<app signing key SHA-256>"
    ]
  }
}]
```

Path on the live site:
```
https://ambitbudget.com/.well-known/assetlinks.json
```

This file must be:
1. **Publicly readable** (not behind auth or geoblocking)
2. **Served with Content-Type `application/json`** (Vercel does this by default)
3. **NOT redirected** (a 301 to `www.ambitbudget.com` will break validation —
   make sure the apex domain serves the file directly)

Quick verification once it's deployed:
```powershell
Invoke-WebRequest -Uri "https://ambitbudget.com/.well-known/assetlinks.json" -UseBasicParsing
```
Should return status 200 with `Content-Type: application/json` and the JSON
body above.

## Migration path: PWABuilder TWA → native Tauri Android

When Tauri 2's Android target matures enough to be the production build,
switching is non-breaking as long as ALL of the following are preserved:

1. **applicationId**: stays `com.kifolloyair.ambitbudget`.
2. **versionCode**: monotonically increases. The TWA-via-PWABuilder build
   ships as versionCode 1+. The first native Tauri build must use a higher
   versionCode than the latest TWA build, never the same or lower.
3. **Upload key**: the same `.keystore` from PWABuilder is used to sign the
   native Tauri `.aab`. Tauri's Android build supports configuring a custom
   keystore via `gradle.properties`. The upload key is what Play matches on
   — if the keystore is lost, the app cannot be updated, period.
4. **App Signing key**: managed by Play. Survives any developer-side change.
5. **assetlinks.json**: only needed for TWA. Native builds can leave the
   file in place (harmless) or remove it.

If the keystore IS ever lost, Play supports a "request upload key reset"
flow (Play Console → Setup → App integrity → Reset upload key). Google
takes 1–2 business days to approve and emails a fresh upload certificate.
Do not rely on this — back up the keystore to a password manager and an
encrypted offsite location.
